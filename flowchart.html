User Opens GeoGuard

Choose Auth Method

Email/Password Registration OR Google OAuth 2.0

Email Path

📧 Email Verification

OTP sent via SMTP → stored in Redis (10 min TTL) → user submits OTP

Google Path

🔑 Google OAuth

Passport.js redirects to Google → callback creates/finds user

🔒

Password Hashing & User Creation

bcrypt hashes password → User document saved to MongoDB

🎫

JWT Token Generated

jsonwebtoken signs { sub: userId } → sent to frontend

📬

Welcome Email Sent

Async email via Nodemailer (fire-and-forget)

🗺️

Map Dashboard Loads

Zustand stores token → Socket.IO connects with JWT → nearby incidents fetched

📝

User Opens Report Panel

Multi-step form: category (10 types) → title → severity (1-5) → description → media

📍

Location Captured

GPS auto-fill OR map click-to-place → address reverse-geocoded

☁️

Media Upload (Cloudinary)

Images/videos uploaded → URLs + thumbnails stored

🔍

Text Validation (Gemini AI)

Synchronous spam/gibberish check — blocks fake reports instantly

❓

Is Text Valid?

AI checks for spam, gibberish, and irrelevant content

✅ Valid

Incident Created

Saved to MongoDB with 2dsphere index → +10 points to user → Redis cache invalidated

❌ Spam

Report Rejected

400 error returned with reason — report not saved

📡

Socket.IO Broadcast

io.emit('new_incident') → all connected clients see it on map instantly

🧠

Background AI Verification

Async Gemini analysis: confidence score, risk level, AI tags, sensitivity flag

🤖

AI Verdict

Is the incident genuine or fabricated?

✅ Genuine

Enriched with AI Data

aiConfidence, riskScore, riskLevel, aiTags saved → update broadcast

🚫 Fake

Marked Fake & Deactivated

isActive=false → user loses 10pts + 20 trust penalty → removed from map

🔌

Socket.IO Connection

Client connects with JWT token in handshake → server authenticates via middleware

🗺️

Join Geographic Room

Client emits 'join_area' with {lng, lat} → joins room like "area:77:28"

📍

Location Tracking

'update_location' → User.lastLocation updated → stored in userLocations Map

⏱️

Danger Zone Check (30s interval)

Server queries high-severity incidents (≥4) from last 2 hours

📏

Haversine Distance Check

Is user within 800m of a high-risk incident?

⚠️ In Danger Zone

Alert Emitted

'danger_zone_alert' → "⚠️ High-risk incident 350m away: ..."

✅ Safe

No Alert

User continues normal operation
